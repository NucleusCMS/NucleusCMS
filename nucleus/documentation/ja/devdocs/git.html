マージ

masterブランチへのマージ

操作手順
  1. 作業ブランチをmasterブランチにします
    git checkout master
  2. マージ作業を開始します
    git merge --no-commit --no-ff -X theirs v3.8dev
  3. 競合を解決します
    TortoiseGit: 競合を解決 :
    クリックして、移動などで削除されていないファイルを削除を選択します
  4. 作業フォルダとv3.8devを比較します
    TortoiseGit: ビジョングラフ,refブラウザ : v3.8dev : [作業ツリーと比較]
    変更が残っていれば クリックして変更部分を確認し、変更を取り消すなどして 保存します。
    必要な変更があった場合は、v3.8devへ後ほど適用するかマージを中止するか検討しましょう。
  5. コミットします。
    コマンド、GUIお好きな方法でコミットしてください。
  6. 最後にもう一度　同一かどうか確認します
    git diff --name-only v3.8dev master
    git diff v3.8dev master
